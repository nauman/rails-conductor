Cloudflare R2 integration

Monitor and manage R2 object storage from Conductor — buckets, usage, and credentials — so app storage and backups have one home in the control plane, from UI and over MCP.

Pillar: Provider automation Priority: P1 Effort: M Status: Planned

Problem

Apps use R2 for Active Storage and backups, but Conductor has no R2 surface — buckets are created and credentialed by hand in the Cloudflare dashboard, with no visibility into usage or which app owns which bucket. The control plane should own this like it owns servers and databases.

Current state

Backups can target R2 (docs/plans/backups-r2.md) and R2 is named across the vision, but there's no R2 client, no bucket model, and no UI/MCP tool. Credentials live in env/vault, not managed by Conductor.

Proposed design

An R2Client over the Cloudflare R2 / S3-compatible API, an R2Bucket model linked to org (and optionally app), a UI panel (buckets, size, object count, last activity), and MCP tools (conductor_storage: list / create / credentials / lifecycle). Credentials resolved via the secret source (vault), never persisted plaintext — consistent with secretless deploys.

Implementation slices

  1. R2Client + R2Bucket model (org/app-scoped); store the account + access keys as managed credentials.
  2. Monitor: UI panel + a read tool — buckets, usage/size, per-app ownership.
  3. Manage: create bucket, issue/rotate scoped credentials, set lifecycle rules; MCP conductor_storage tool.
  4. Wire app Active Storage + the backup target to a managed bucket (one-click "give this app R2 storage").

Acceptance criteria

  • From UI and MCP, an operator sees all R2 buckets with usage and can create one + scoped credentials.
  • An app can be pointed at a managed bucket for storage/backups without leaving Conductor.
  • R2 credentials are resolved from the secret source, never stored plaintext.

Dependencies & related

Pairs with backup restore (R2 is the backup store) and the secret source from secretless deploys. First of the Connected Services hub in VISION.md.
